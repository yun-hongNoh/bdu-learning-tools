기본 상태에서는 해결 구조를 보여주지 않습니다. 먼저 굵어야 함과 가늘어야 함의 충돌만 확인합니다.

칫솔모에서 다른 문제로 확장해 봅시다

아래 6개 사례는 모두 “하나의 특성 = 서로 반대되는 요구” 구조입니다. 각 사례에서 원리별 해결 방식이 어떻게 다른지 구분해 보세요.

교재 두께 문제
특성: 교재 두께
모순: 충분한 설명 ↔ 휴대·반복 학습
전체/부분 분리 핵심 개념은 본문에 두고, 사례·보충자료는 부록으로 나눕니다.
공간 분리 종이 교재에는 필수 내용만 두고, 추가 자료는 QR·웹 공간으로 분리합니다.
항아리 냉장고 문제
특성: 냉각 방식
모순: 내부는 시원해야 함 ↔ 에너지 사용은 낮아야 함
조건 분리 물의 증발과 통풍이 가능한 조건에서 냉각 효과가 발생하게 합니다.
공간 분리 내부 저장 공간과 외부 증발·통풍 공간의 역할을 나눕니다.
마을버스 활성화 문제
특성: 운행 빈도
모순: 자주 다녀야 함 ↔ 운영 비용은 낮아야 함
시간 분리 출퇴근·등하교 시간에는 집중 운행하고, 낮은 수요 시간에는 간격을 늘립니다.
조건 분리 이용자 수, 요일, 행사 여부에 따라 정규·예약형 운행을 다르게 적용합니다.
스마트폰 화면 크기
특성: 화면 크기
모순: 크게 봐야 함 ↔ 작게 휴대해야 함
시간 분리 사용할 때는 펼치고, 이동할 때는 접어 작은 상태로 둡니다.
전체/부분 분리 기본 본체는 작게 두고, 보조 화면이나 확장 패널이 일부 기능을 맡습니다.
강의 영상 길이
특성: 영상 길이
모순: 충분히 설명하려면 길어야 함 ↔ 집중하려면 짧아야 함
전체/부분 분리 전체 강의를 핵심·심화·실습·요약 영상으로 나누어 제공합니다.
시간 분리 수업 전에는 핵심 예습, 수업 중에는 실습, 수업 후에는 요약 복습을 제공합니다.
팀 프로젝트 발표자료
특성: 정보량
모순: 근거는 많아야 함 ↔ 화면은 단순해야 함
공간 분리 핵심 메시지는 본 슬라이드에, 세부 근거는 부록·QR·발표자 노트에 둡니다.
전체/부분 분리 전체 보고서는 충분히 유지하되, 발표 화면은 핵심 부분만 보여줍니다.

내 프로젝트 적용

프로젝트 적용 단계에서는 중앙에 6개 전이 사례가 표시됩니다. 오른쪽에는 내 프로젝트 문제를 정리하는 작성 프롬프트가 나타납니다.
교재 두께 문제
교재 두께 = 두꺼워야 함 ↔ 얇아야 함
추천 원리: 공간 분리 / 전체·부분 분리
핵심은 종이, 부가 자료는 QR·웹·부록으로 분리합니다.
항아리 냉장고 문제
냉각 구조 = 차가워야 함 ↔ 에너지 사용은 낮아야 함
추천 원리: 조건 분리 / 공간 분리
습도·통풍 조건과 증발 냉각 공간을 분리해 활용합니다.
마을버스 활성화 문제
운행 빈도 = 높아야 함 ↔ 운영 비용은 낮아야 함
추천 원리: 시간 분리 / 조건 분리
출퇴근 시간에는 집중 운행하고, 수요가 낮을 때는 축소합니다.
스마트폰 화면 크기
화면 크기 = 커야 함 ↔ 작아야 함
추천 원리: 시간 분리 / 전체·부분 분리
볼 때는 펼치고, 이동할 때는 접거나 보조 화면을 분리합니다.
강의 영상 길이
영상 길이 = 길어야 함 ↔ 짧아야 함
추천 원리: 시간 분리 / 전체·부분 분리
전체 강의는 유지하되, 핵심 영상과 심화 영상으로 나눕니다.
팀 프로젝트 발표자료
정보량 = 많아야 함 ↔ 적어야 함
추천 원리: 공간 분리 / 전체·부분 분리
핵심 슬라이드는 단순하게, 세부 근거는 부록·QR·발표자 노트로 분리합니다.
